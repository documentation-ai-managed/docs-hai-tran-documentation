  <Expandable id="content-tips" title="What are the best practices for content management?">

    Organize docs with clear hierarchies. Use frontmatter for metadata and components for rich layouts.

    <Steps>
      <Step title="Structure Pages" icon="file-text">
        Start every page with YAML frontmatter:

        ````markdown
        ---
        title: Your Page
        description: Brief summary
        ---
        ````

        Follow with H2 headings.
      </Step>
      <Step title="Use Components" icon="layers">
        Incorporate `<Callout>`, `<Steps>`, and `<Tabs>` for engagement.
      </Step>
      <Step title="Version Control" icon="git-branch">
        Connect to GitHub for seamless updates.
      </Step>
    </Steps>

    <Callout kind="success">
      Preview changes live to catch issues early.
    </Callout>

  </Expandable>

  <Expandable id="troubleshooting" title="How do I troubleshoot common errors?">

    Follow this process for most issues.

    ```mermaid
    graph TD
        A[Identify Error] --> B[Check Logs]
        B --> C{API Related?}
        C -- Yes --> D[Verify `YOUR_API_KEY`]
        C -- No --> E[Clear Cache]
        D --> F[Retry Request]
        E --> F
        F --> G[Resolved?]
        G -- No --> H[Contact Support]
    ```

    <Steps>
      <Step title="Parsing Errors" icon="bug">
        Ensure you escape special characters like `{variable}` and `<Component>` with backticks in prose.
      </Step>
      <Step title="Build Failures" icon="terminal">
        Run locally first:

        <CodeGroup tabs="npm,pnpm">
          ```bash
          npm run build
          ```
          ```bash
          pnpm build
          ```
        </CodeGroup>
      </Step>
      <Step title="Component Issues" icon="alert-circle">
        Verify PascalCase and double quotes: `<Callout kind="info">Text</Callout>`.
      </Step>
    </Steps>

  </Expandable>

</ExpandableGroup>
