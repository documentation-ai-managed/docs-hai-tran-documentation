## Platform Overview

Hai Tran Documentation serves as your central hub for organizing project docs. You create structured spaces, collaborate with teams, and maintain up-to-date guides. Use it to document APIs, user flows, deployment processes, and more. The platform supports MDX for rich, interactive content with components like cards, steps, and tabs.

## Getting Started

Follow these steps to set up your first space.

<Steps>
  <Step title="Create Account" icon="user-plus">
    Sign up at `https://app.haitran-docs.com` with your email.
  </Step>
  <Step title="New Space" icon="plus">
    Click "New Space" and name it for your project, like "API Docs".
  </Step>
  <Step title="Add Pages" icon="file-text">
    Create `introduction.mdx` and use the MDX editor.
  </Step>
  <Step title="Publish" icon="globe">
    Hit publish to share your live docs.
  </Step>
</Steps>

## Brand Color Guidelines

Incorporate the brand color `#3B82F6` consistently. Apply it to headings, buttons, and accents for a unified look.

<Tabs>
  <Tab title="CSS" icon="css3">
    Define the primary color in your styles.

````css
:root {
  --brand-primary: #3B82F6;
}

.button {
  background-color: var(--brand-primary);
  color: white;
}
````
  </Tab>
  <Tab title="Tailwind" icon="tailwind">
    Use the custom color in Tailwind config.

````javascript
module.exports = {
  theme: {
    extend: {
      colors: {
        primary: '#3B82F6',
      },
    },
  },
};
````

    Then: `<div className="bg-primary text-white">Button</div>`
  </Tab>
  <Tab title="JS Usage" icon="javascript">
    Dynamically apply the color.

````javascript
const brandColor = '#3B82F6';

document.documentElement.style.setProperty('--brand-color', brandColor);
````
  </Tab>
</Tabs>

<Expandable title="Advanced Branding" default-open="false">
  Customize further with CSS variables. Override defaults in your theme file:

  ```css
  :root {
    --brand-primary: #3B82F6;
    --brand-secondary: #1D4ED8;
  }
  ```

  Test responsiveness: Ensure colors pass WCAG AA contrast ratios.
</Expandable>
